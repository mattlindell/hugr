# ADR 0004 — HNSW index, bge-m3 text embeddings at 1024 dimensions

**Status:** Accepted — 2026-07-12 (embeddings apply to the **1.0** episodic tier, not 0.5)

## Context

The episodic `memories` tier uses vector similarity. The donor ships an **IVFFlat** cosine index
and `all-mpnet-base-v2` (768-dim) embeddings computed inside the MCP container. hugr routes all
inference through the user's **LiteLLM** gateway (`192.168.20.202:4000`) on the Jetson AGX Orin.
The only embedder currently deployed there is `jina-code-embeddings` — **code-tuned**, reserved for
Vera, and wrong for natural-language memories. A reranker `bge-reranker-v2-m3` is already deployed.

## Decision

- **Index: HNSW** (cosine, `<=>`), replacing IVFFlat — better recall/latency, no training step.
- **Embeddings: `bge-m3` at 1024 dimensions**, deployed to the Jetson `jetson-inference` stack via
  the existing `home-server/llama-cpp` GGUF pattern and routed through LiteLLM. Chosen over
  `nomic-embed-text`/768 because it is the sibling of the already-deployed `bge-reranker-v2-m3`
  (coherent embed+rerank pair), a stronger retriever, and reuses a proven deploy path. Schema
  column is `vector(1024)`; `halfvec` is unnecessary at this dimension.
- **Reranker (optional in 1.0):** `bge-reranker-v2-m3` for ANN over-fetch → rerank. Note it returns
  **raw cross-encoder logits (~+9…−11), not sigmoid 0–1** — any thresholding must use the logit
  scale.
- Embeddings run **on the Jetson via LiteLLM**, not host-side — this keeps `torch` /
  `sentence-transformers` out of the desktop CLI environment entirely, reinforcing the Cold-start
  Invariant (ADR 0003).

## Consequences

- Facts are **not embedded** — this ADR is irrelevant to the 0.5 fix and entirely scoped to 1.0.
- bge-m3 is the one remaining **inference-side prerequisite** for 1.0 and is a **GitOps** change to
  the home-server repo (Reconciler-applied, not hand-applied).
- The CLI becomes a thin HTTP embeddings client; no local ML stack.
- Distillation uses the already-live `ollama/qwen2.5-14b-32k` — no additional model deploy needed.
